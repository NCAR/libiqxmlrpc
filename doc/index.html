About

Libiqxmlrpc is an object-oriented library, which represents simple XML-RPC solution both for client and server sides.

Main goal of project is to provide:

  • Nice object oriented design;
  • Maximum separation of code which operates with XML-RPC structures from that part, which provides transport functionality;
  • HTTP, HTTPS as standart transports;
  • Comfortable way to extend library's functionality.

Language: C++

Platforms:

  • Linux, Solaris;
  • Cygwin (client's code works well);
  • Win32 (client only).

Dependencies

You need modern enough C++ compiler, e.g. GNU g++ of version 3.2 and above.

Libraries required:

  • libxml++-1.0
  • OpenSSL

Features

XML-RPC support:

  • Basic XML-RPC standard support;
  • Support for Nil values.

Other features:

  • HTTP and HTTPS server transports (over non-blocking socket);
  • HTTP and HTTPS client transports (over non-blocking socket);
  • Serial and multithreading server method's execution functionality;
  • Support for server's introspection;
  • FCI conformance;
  • Support for transparent charset conversion.
  • Support for persistent connections.
I am trying to make library classes to be STL-compatible where it possible. I.e. to use similar method names when I want to express similar semantic as well as make to some classes compatible with standard algorithms.
